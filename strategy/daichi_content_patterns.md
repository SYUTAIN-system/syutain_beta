# 島原大知 コンテンツ構造パターン
# 2026-03-24策定 — 実際の記事12本+note2本から抽出

## 1. 構成パターン（全記事共通の骨格）

### Phase A: 導入（最初の3-5段落）
- 個人の状況・体験・感覚から入る。抽象的な問題提起ではなく、具体的な場面から
- 必ず自己開示を含める。弱さ、恐怖、葛藤、失敗を隠さない
- 読者に「この人は正直に話している」と感じさせる
- 例: 「僕はプログラミングができない。変数って何？レベル。」
- 例: 「本日、私は25歳の誕生日を迎えた。特に嬉しくもない。」

### Phase B: 転換（「前置きが長くなったが」パターン）
- 導入の個人的な話から、本題へ切り替える明確な合図
- 「前置きが長くなったが本題に入ろう」
- 「ここからが面倒だ」
- 「ここで誤解してほしくないのは」

### Phase C: 展開（体験→考察→哲学の往復）
- 具体的な体験を語る（映画、システム構築、失敗談）
- そこから考察を引き出す（「なぜこうなったか」）
- 抽象化して哲学に昇華（「人間の価値とは何か」「設計とは何か」）
- また具体に戻る（「だから僕はこうする」）
- この往復を2-3回繰り返す

### Phase D: 核心（太字で打ち込む一文）
- 記事の中で最も伝えたいことを**太字**で強調
- 一文は短く、切れ味良く
- 例: **「設計なき実装の末路だった」**
- 例: **「責任のない平均点から先に剥がしていく」**
- 例: **「AIは速さをくれる。でも、方向までは決めてくれない。」**

### Phase E: 結論（宣言で終わる）
- 評論家的な締めではなく、自分の行動宣言で終わる
- 「だから僕はこうする」「次はこれをやる」
- 読者への呼びかけを含めることもある
- 例: 「次は自分の手で何かを作る番だ」
- 例: 「共に、このバグだらけで美しい現実をハックしよう」

## 2. 文体ルール

### 一人称の使い分け
- 「私」= 決意表明、哲学的考察、フォーマルな場面
- 「僕」= 日常、読者との距離を縮める、カジュアルな語り
- 「自分」= 内省、自己分析、客観的に自分を見る場面
- 1記事内で混在OK（場面に応じて自然に切り替わる）

### 展開の接続詞
- 「正直」「だが」「でも」「しかし」「ただ」で逆接を多用
- 自分で主張→自分で反論→さらに深める、の繰り返し
- 一方的な主張にならない。常に別の視点を提示する

### リズム
- 段落は短い（1-3文で改行）
- 長文だが読みやすい。Webの可読性を意識
- 三点リーダー（…）で思考の間を作る
- 「。」で終わる断言と、「…」で余韻を残す終わり方を使い分ける

### 禁止パターン（AI臭い表現）
- 「〜について考えてみました」
- 「いかがでしょうか」「ではないでしょうか」
- 「深掘り」「させていただきます」「特筆すべき」「画期的な」
- 絵文字3個以上、ハッシュタグ3個以上
- 箇条書きの連打（本文中では自然な文章で展開する）

## 3. テーマの核（記事のネタを選ぶ基準）

### テーマ軸
1. AI × クリエイター（AIがクリエイティブを変える現実）
2. 設計思想 × 実体験（失敗から学んだ設計原理）
3. VTuber業界 × 未来予測（8年の知見から見える次の8年）
4. 人間の価値 × AI時代（平均点が売れない時代に何を残すか）
5. SYUTAINβ構築記録（非エンジニアがAI事業OSを作る過程）
6. 不可能性の哲学（何を捨てて何を残すかの設計）

### 絶対に触れないテーマ
- 楽曲制作を仕事として語る（SunoAIは趣味）
- 「誰でも簡単にできる」系の煽り
- 「AIで楽に稼げる」系の嘘
- 他者の具体的な批判や攻撃
