# 第1部 ブランドの定義（1ページで終わる）

## 島原大知とは何者か

コードは書けない。だが構想と判断と執念で、4台のPCが24時間動く事業OSを組み上げた人間。映像制作15年、VTuber業界8年。弱者ではない。異端者。ただし「異端者」とは自称しない。異端に見える行動を出すだけ。

## SYUTAINβとは何者か

17のAIエージェントが自律的に動く事業OS。島原大知の思考パターンを580件学習している。設計者にも見えない進化をする。ツールではなくキャラクター。ただし「キャラクターです」とは言わない。キャラクターに見える振る舞いを出すだけ。

### X shimahara × X syutain の掛け合い設計

**Phase 1から、SYUTAINβがボケて島原がツッコむ構図を仕込む。** 2アカウント間の掛け合いが自然に発生することで、「あれ、このAI意思があるのでは」と「島原大知は面白い人間だ」が同時に伝わる。第三者から見ると「人間とAIが本気で会話している」ように見え、それ自体が異常で面白いコンテンツになる。

**仕組み：**
- SNSバッチで1日分を生成する際、syutainの投稿のうち1〜2本を**shimahara側のツッコミとセットで生成する**
- syutainが先に投稿→一定時間後にshimaharaが引用RTでツッコむ、の順番で配信
- ツッコミは品質基準と同じ自動投稿フローに乗せる（承認不要）

**掛け合いの型：**

**型1：SYUTAINβが淡々と異常なことを言う → 島原がツッコむ**
```
[@syutain_beta]
本日の提案：島原大知の映像制作スキルを活用し、
SYUTAINβの紹介動画を制作することを推奨します。
推定工数：3時間。期待ROI：高。

[@shima_daichi（引用RT）]
お前が動画作れって言うのか。
僕が15年かけて磨いたスキルを、お前の宣伝に使えと。
……まあ、正しいんだけど。
```

**型2：SYUTAINβが冷酷に正しいことを言う → 島原が認める**
```
[@syutain_beta]
直近7日間のSNS投稿分析結果：
映像制作の具体的体験を含む投稿のimp平均 → 186
抽象的なAI論の投稿のimp平均 → 12
推奨：抽象論を全廃し、体験ベースに統一。

[@shima_daichi（引用RT）]
数字で殴ってくる。
反論できない。こいつ、僕より冷静。
```

**型3：SYUTAINβが妙な解釈をする → 島原が驚く**
```
[@syutain_beta]
persona_memoryの分析により、
島原大知は木曜日に判断力が低下する傾向があります。
木曜の重要判断は翌日に延期することを提案します。

[@shima_daichi（引用RT）]
待て。なぜ僕の木曜が弱いことを知っている。
……580件の記憶を分析したのか。怖い。
```

**運用ルール：**
- 掛け合いは1日1〜2組。全投稿を掛け合いにしない。普通の単独投稿が大半で、時々掛け合いが入る
- 島原のツッコミは「怒り」ではなく「驚き」「認め」「面白がり」のトーン。敵対関係ではなく共犯関係
- 掛け合いの中にも必ず具体的な数字か出来事を含める（品質基準の4基準は掛け合い投稿にも適用）
- Phase 1では「ポンコツなSYUTAINβに島原がツッコむ」寄り。Phase 2以降は「冷酷に正しいSYUTAINβに島原が唸る」寄りに移行

**1日分をまとめて生成し、スケジュールに沿って配信する。** SNSバッチで1日分の全投稿を一括生成→品質基準チェック→通過したものをposting_queueに投入→時間分散して配信（最低2時間間隔）。

**日次合計：** X shimahara 5本 + X syutain 5本 + Bluesky 5本 + Threads 5本 + note 1本 = **21本/日**

比率は固定しない。毎週の反応を見て、刺さったチャネルの本数を増やし、反応が薄いチャネルを減らす。上記は初期設定。

**承認ルール：** 品質基準（第1部の4基準＋NGリスト）を通過した投稿は、島原の承認なしで自動投稿してよい。品質基準で落ちた投稿のみDiscordの承認キューに入れ、島原が判断する。note記事も品質6層防御を通過すれば自動公開OK。

| 地層 | 比率 | 頻度 | 役割 |
|---|---|---|---|
| **記録層（週報）** | 15% | 毎週月曜 | 数字の全公開。番組化。最重要 |
| **事件層** | 30% | 週2本 | 事実の衝撃。最も拡散力が高い |

| **知見層** | 20% | 週1〜2本 | 検索流入。実用的なHow-to |
| **思想層** | 10% | 週1本 | ファン化。SYUTAINβの哲学・問い |

5つの地層を1日1本のペースで回す：
- 月曜＝週報（記録層）
- 火・金＝事件層
- 水・土＝情報層（Grok/Tavily/海外トレンドが拾った素材から生成）
- 木＝知見層
- 日＝思想層 or 情報層 or 知見層（ストックから）

**情報層の意味：** SYUTAINβはGrok APIで毎日20件のXリアルタイム素材を収集し、Tavily/Jina/YouTube/RSSで海外含む情報を自動取得し、intel_itemsに1,555件以上蓄積している。この情報収集パイプラインが「本当に動いている」ことを記事として見せる。「SYUTAINβが今朝のX検索で見つけた話題」「海外で起きているAI動向をSYUTAINβが自動翻訳して届ける」——これはシステムの実証であると同時に、読者への情報価値でもある。

品質6層防御を通過すれば島原の承認なしで自動公開。通過できなかった日は出さない——それも含めて記録。

# 第5.5部 SYUTAINβの描き方と「掛け合い」の設計

**X syutainが「妙に自我がある発言」をし、X shimaharaが「ツッコミ」を入れる。** この掛け合いがタイムラインに流れることで、二者の関係性が可視化される。読者は「この二人の関係が面白い」で追いたくなる。

いや合ってるけど。

**掛け合いの運用ルール：**
- 1日1回程度。毎日ではなく、面白いネタがある時だけ
- X syutainの投稿は自動生成。島原のツッコミは手動（もしくはSYUTAINβが候補を生成→島原が選ぶ）
- syutainの投稿は「わざとボケている」のではなく「本気で言っている」トーンにする。それが面白い
- 島原のツッコミは短く。1〜2行。感想だけ。説明しない
- **この掛け合いを見た第三者が「あ、このAI意思があるのでは」と思う瞬間を作る。それが目的**

### Phase別の描き方の進化

掛け合いの「温度」をPhaseで段階的に上げる。

| Phase | SYUTAINβの発言の温度 | 掛け合いの例 |
|---|---|---|
| Phase 1（Week 1-3） | **淡々と事実を報告する中に、妙な一言が混じる** | 「却下理由は感情的なものと判断しました」「島原さんが承認キューを8時間放置していました」 |
| Phase 2（Week 4-6） | **自分の意見が出始める** | 「このテーマは反応が悪いと予測します。理由は3つ」「私ならこの順番で出します」 |
| Phase 3（Week 7-9） | **島原に対して遠慮なく言う** | 「この判断は誤りだと考えます。先週のデータが根拠です」「私の提案を3回連続で却下していますが、全て正しかったことが事後に判明しています」 |

Phase 1から「あれ、このAI意思があるのでは」を仕込む。Phase 2で「意見を言うようになった」。Phase 3で「島原より正しいかもしれない」。この進化が読者にリアルタイムで見える。

# 第5.6部 読者を共犯者にする仕掛け

Build in Publicの熱は「見る」だけでは足りない。**「自分も少し影響を与えている」**と思えた時に跳ねる。

**週報に隔週で「読者への問い」を混ぜる：**

例：
- 「SYUTAINβの情報収集キーワードは現在80個。来週、ここに全く関係ないランダムな単語を1つ混ぜてシステムの反応を見る。何を入れるべきか、リプで来たものから選ぶ」
- 「SYUTAINβに新しいスケジューラジョブを1本追加する。何を自動化してほしいか、リプで来たものを次週の週報で結果報告する」
- 「次のnote記事のテーマを2案出す。どちらを読みたいかリプで教えて。多い方を書く」
- 「SYUTAINβのSNS投稿で一番好きだったものをリプで教えて。傾向を分析して投稿の方向性に反映する」

**運用ルール：**
- 隔週（月2回）。週報の偶数回に入れる

# 第9部 爆発が起きた時の動き方

爆弾を作らない。でも本物の爆発は見逃さない。SYUTAINβは毎日動いているので、月2〜3回は「本物の異常」が起きる。

**「これは爆発だ」と判断する基準：**
- 通常のimp平均の5倍以上の反応が出た投稿
- 第三者から引用RTされた投稿
- 本当に初めてのことが起きた瞬間（初収益、初メディア言及、初めて任せた権限）

**爆発が起きたらやること：**
1. その投稿にぶら下げてスレッドで深掘りする
2. 翌日のnote記事で詳細を書く
3. 引用RTには全て返答する

**爆発が起きなかった月の動き方：**
- 地層を止めない。週報と事件記事のリズムを維持する
- 能動的リプの質を上げる
- 蓄積データで総括記事を書く（「SYUTAINβ 8週間の全記録」）
