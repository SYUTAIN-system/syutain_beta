# 承認ポリシー（ApprovalManager準拠）

## 人間承認が必須な操作

### Tier 1: 即座に承認要求（自動実行禁止）
- SNS投稿（X / Bluesky / note）
- 商品公開・価格設定（Booth / Stripe）
- 暗号通貨の売買注文
- 月額予算の変更
- 新規APIキーの登録

### Tier 2: 自動実行可能だが事後通知
- 情報収集パイプラインの実行
- LLMモデルの切り替え
- タスクの自動再スケジュール
- ノード間タスク再分配

### Tier 3: 完全自動（通知なし）
- ヘルスチェック
- ログローテーション
- JetStreamストリーム管理

## 承認フロー
1. ApprovalManagerがDiscord Webhook + Web UIで通知
2. Web UIのチャットインターフェースで承認/却下
3. タイムアウト: 24時間（超過時は自動却下）
4. 却下時: 理由推測 → 代替案を即座に提示
